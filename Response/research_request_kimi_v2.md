# Folunar — 第二轮综合咨询请求

## 项目背景

"从零开始自主 AI Agent"项目。核心理念：
- **"思维 = 概率 + 最优选择"** — 人类和机器认知共享同一底层机制
- **Brain (意图分类) + Hand (参数执行)** 分离架构
- 22M frozen Sentence Transformer + 4K param head = Brain，外部 LLM / 模板引擎 = Hand
- CPU-only（Intel Core Ultra 9 185H, 30GB RAM）
- Docker 沙箱运行

### ✅ Brain Necessity Benchmark
18 个真实 Linux 任务 × 3 种策略:

```
              随机     关键词   训练分类器
L1 (单步)     60%      60%      70%  🟡 +10pp
L2 (两步)     80%      60%      20%  🔴
L3 (多步)     67%      67%      67%  🟡 ±0
总计          67%      61%      56%
```

**关键发现**:
1. 分类器在 L1 单步任务上确实优于随机 (+10pp)，验证了 Brain 有基本信息价值
2. L2 失败的原因: 分类器只看单步提示，没有全局任务上下文
3. 18 个任务统计上不足以区分策略间差异
4. 参数的确定性默认值比意图选择本身更限制成功率

## 三条待办路径

我们面临三条都做、但需要排优先级的路线:

### 🔵 A) 继续搭 v2 闭环系统
完成 "编码 → 大脑 → 手 → Docker → RND → 在线学习" 闭环
- 统一状态编码器 (位置编码+文件状态+历史)
- RND 好奇心模块 (预测误差 → 探索奖励)
- 闭环集成测试
- 风险: 如果 Hand 参数生成还不可靠，闭环意义有限

### 🟢 B) 改进分类器精度
65% 不够高，需要 80%+
- 更多/更好的数据（ChatGPT 强调的"数据质量先于架构"）
- 注意力更强的架构 (BiLSTM head, 不只用 Linear)
- 解决 INSPECT/HELP 混淆
- 为 L2+ 任务设计带"任务上下文"的输入格式

### 🟡 C) 解决参数生成
本地 LLM (Qwen 0.8B/DeepSeek 1.5B/Gemma 8B) 全部无法做参数推理
- 模板引擎已搭建 (8 个 intent templates, 白名单路径)
- 需要: 从任务描述到参数的确定性/半确定性映射
- 或者: 找能工作的本地模型 / 轻量 prompt 策略
- 瓶颈卡在这里就会影响整个闭环
