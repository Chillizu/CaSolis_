# CaSolis_ v2 — ModelCluster 路线规划

## 核心理念

不是「一个模型会所有事」，是「多个专家 + 一个路由器学会选谁」。

```
Input → 路由器 → 选哪个专家 → 专家输出 → 共享头 → Token

路由器本身不大（6.9M），但每个专家可以是完整的 Mamba block。
总参数量可以上 50M-100M，但每次只激活一个专家，推理和训练成本不变。
```

## 架构

```
                      ┌─────────────────────────────┐
                      │      Shared Embedding        │
                      │        nn.Embedding(V, d)     │
                      └──────────────┬──────────────┘
                                     ↓
                      ┌─────────────────────────────┐
                      │      Router (6.9M)           │
                      │  当前状态 → 选哪个专家？       │
                      │  输入: embed + 语境状态         │
                      │  输出: 4 个专家的选择概率        │
                      └──┬────┬────┬────┬────────────┘
                         │    │    │    │
                    ┌────┴┐ ┌┴────┴┐ ┌┴────┐
                    │E0   │ │E1    │ │E2   │ │E3
                    │命令  │ │创造  │ │世界  │ │评估
                    │专家  │ │专家  │ │专家  │ │专家
                    │Mamba │ │Mamba │ │Mamba │ │Mamba
                    │12.6M │ │12.6M │ │12.6M │ │12.6M
                    └──┬──┘ └──┬───┘ └──┬──┘ └──┬──┘
                         │    │    │    │
                         └────┴────┴────┘
                              ↓
                      ┌─────────────────────────────┐
                      │      Shared Heads            │
                      │  ┌ lm_head (token 预测)       │
                      │  ├ world_head (输出预测)       │
                      │  └ confidence (质量判断)       │
                      └──────────────┬──────────────┘
                                     ↓
                                   Token
```

### 专家职责

| 专家 | 角色 | 初始化 | 训练目标 |
|------|------|--------|---------|
| E0 Commander | 生成常规命令 | big-mamba checkpoint | 标准 LM loss |
| E1 Creator | 生成新奇命令 | big-mamba + 噪声 | LM loss + 多样性奖励 |
| E2 World Predictor | 预测输出结果 | big-mamba (世界头) | 嵌入 MSE loss |
| E3 Evaluator | 评估命令好坏 | big-mamba | 置信度 loss + 结果验证 |

### 路由器

不是大模型。是一个小 MLP：
```
输入: embed(token) + one-hot(上次选谁) + 语境特征
隐藏: 128 → 64
输出: 4 个专家的 softmax 概率
参数: ~100K
```

路由器学会：
- 常规情况 → E0（命令专家）
- 好奇心高 → E1（创造专家）
- 需要预测 → E2（世界专家）
- 需要检验 → E3（评估专家）

## 训练策略

### 阶段 1: 搭建（今天就能做）

```
1. 写 ModelCluster 代码
   - 4 × MambaBlock(1024) + 共享 embed + 路由器 + 共享头
   
2. 从 checkpoint 初始化
   - E0 ← big-mamba/model_best.pt (LM head)
   - E2 ← big-mamba/model_best.pt (World head)
   - E1 ← same + 初始化噪声
   - E3 ← same (自信头从头训)

3. 预测试：确保前向/反向/生成都正常
```

### 阶段 2: 联合训练（今晚 / 明天）

```
1. 所有专家 + 路由器一起训
   使用 load-balancing loss（防止所有任务都被 E0 抢走）

2. 数据：1500 条 + 在线生成（runner 不断产生新数据）

3. 训练目标：
   ┌ LM loss（所有专家的 token 预测）
   ├ World loss（E2 的输出预测）
   ├ Confidence loss（E3 的置信度校准）
   ├ Router load-balancing（各专家使用率均衡）
   └ Router accuracy（路由器选对了吗？）
```

### 阶段 3: 自主运行

```
1. Docker 沙箱 + ModelCluster
2. 路由器每步选专家 → 专家生成命令 → 执行 → 观察
3. 世界专家预测结果 → 如果误差大 → 创造专家被激活
4. 评估专家判断输出价值 → 有用的存起来继续学
```

## 参数量估算

| 组件 | 参数 | 说明 |
|------|------|------|
| Shared Embed | 2.1M | 2041 × 1024 |
| Router | 0.1M | Linear stacks |
| E0 Commander | 12.6M | MambaBlock(1024) |
| E1 Creator | 12.6M | MambaBlock(1024) |
| E2 World | 12.6M | MambaBlock(1024) |
| E3 Evaluator | 12.6M | MambaBlock(1024) |
| Shared Heads | 3.1M | lm_head + world_head + conf |
| **Total** | **~56M** | 但每次只激活 ~15M |

## 和之前方案的本质区别

| 之前 | 现在 |
|------|------|
| 1 个模型要会所有事 | 4 个专家各司其职 |
| 瓶颈在单个模型容量 | 瓶颈在路由器质量 |
| 增加能力 = 加大模型 | 增加能力 = 加专家 |
| 创造力 ≈ 噪声 | 创造力 ≈ 切换到创造专家 |
| 死路：训 30 个 epoch 没用 | 活路：路由器越选越准 |

## 项目的四个目标 × 这个架构

| 目标 | 实现 |
|------|------|
| 因果推理 | 世界专家预测输出 → 发现因果模式 |
| 多步规划 | 路由器学会「先查 help 再执行」序列 |
| 好奇心驱动 | 世界专家误差大 → 路由器选创造专家 |
| 泛化 | 新命令 → 类比到已知专家 → 路由器决策 |

---

**核心**: 这不只是「换架构」，是换范式——从「一个模型思考」到「一群模型协作思考」。
