# 四大模型评审消化

> 来源: ChatGPT · Claude · Gemini · Kimi
> 对 `PLAN_BRAIN_HAND.md` 的独立评审

---

## 共识点（所有模型都说一样的）

### 1. "20 个意图不够用"
**"这是一个状态机，不是自主思维"** — Kimi

20 个离散意图无法覆盖真正的探索行为。EXPLORE 只是一个标签，本身不携带探索信息。

### 2. "2000 条数据训 46M 模型不可行"
**"参数/数据比 23,000:1，必然过拟合"** — Kimi

46M 参数的模型在 2000 条数据上从零训练是不现实的。即使有正则化，效果可能不如 sklearn 逻辑回归。

### 3. "Qwen 1.5B 做模板填充是杀鸡用牛刀"
**"Qwen 的工作应该是生成参数，不是翻译模板"** — Claude

除了 EXPLORE 之外，19/20 个意图都是确定性模板（`cat {path}`、`ls {flags} {path}`），根本不需要 LLM。Qwen 唯一不可替代的工作是**生成参数值**（path 应该是什么？pattern 从哪里来？），但计划书里完全没写这部分怎么处理。

### 4. "好奇心驱动会掉进 Noisy TV 陷阱"
**"date, ps aux, uptime 每次输出都不一样，世界模型永远预测不准"** — all

这是 Pathak 2017 已知的失效模式。环境中有固有随机性（时间、进程 PID、系统状态）的输出会被误判为"有趣"，模型会反复探索这些零信息量的源。

### 5. "离线训练 5 epoch ≠ 持续学习"
**"好奇心驱动的是'行为选择'，不是'权重更新'"** — Claude

没有在线学习机制，模型训完就固定了。"持续学习"在这个架构里只是标签，不是事实。

## 各模型独到见解

| 模型 | 最尖锐的批评 | 最有价值的建议 |
|------|-------------|---------------|
| **Kimi** | "这不是从零构建思维，这是在已有 AI 基础设施上搭一个决策-执行的分层系统" | 24h MVP: 用 Sentence Transformer + 硬编码模板，验证核心假设 |
| **Gemini** | "你的 46M 大脑没有在思考，它只是在一个有 20 个按钮的遥控器上按键" | Ensemble World Models 来分离 epistemic/aleatoric uncertainty |
| **ChatGPT** | (提供完整时间线和 KPIs)  | 给出了项目时间表、A/B 测试方案、安全合规评估 |
| **Claude** | "生成难的部分（参数/槽位填充）被藏进架构图的箭头里了" | RND 替代 ICM; 先跑 TF-IDF baseline 再决定要不要 46M; shell 注入警告 |

## 如果现在重写计划，我会怎么改

综合四家意见，新方案应该是：

```
保留:
  ✅ 大脑+手脚分离的方向
  ✅ 世界模型预测作为好奇心信号

砍掉:
  ❌ 20 个硬编码意图 → 改为意图动态生成
  ❌ 46M 从零训练 → 改为预训练 backbone + 轻量头
  ❌ Qwen 做模板填充 → 改为确定性模板引擎
  ❌ 原始 ICM（预测误差） → 改为 RND（固定随机网络目标）
  ❌ 离线训 5 epoch → 改为在线增量学习

保留但重新设计:
  🔄 好奇心驱动的探索 → 用 RND 解决 Noisy TV
  🔄 手脚的 LLM → 只负责生成参数值（path, pattern 等）
  🔄 世界模型 → 预测 RND 目标嵌入，不是原始输出嵌入
```
