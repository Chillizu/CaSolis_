# Stream Architecture v0.1 — 意识流架构

## 核心理念

现有 LLM 是 **被动处理机**：等待输入 → 一次性处理 → 输出 → 休眠。

Stream 架构是 **持续意识流**：永远在生成，永远在处理。它的"静息状态"就是思考。

```
LLM:      输入 ─→ [计算] ─→ 输出 ─→ 结束
Stream:   [思考] ─→ [行动] ─→ [观察] ─→ [思考] ─→ [行动] → ...
```

## 设计原则

### 1. 循环核心 (Recurrent Core)
不是 transformer 的前馈架构，而是 RNN/SSM 式的循环架构。
- 维护一个持续更新的隐藏状态 `h`
- 每个时间步：`h_t = f(h_{t-1}, obs_t, action_{t-1})`
- 状态永不过期，永远在线

### 2. 好奇心即损失 (Curiosity as Loss)
不是外部奖励，而是内置的预测误差驱动。

```
World Model: 预测下一个观察值 obs_{t+1} = predict(h_t)
好奇心:      curiosity = ||predicted - actual||²
学习信号:    模型同时追求两件事：
  - 准确预测常规事务（低预测误差）
  - 主动探索意外事务（高预测误差 → 好奇）
```

### 3. 行动即表达 (Action as Expression)
模型不需要解析 XML。它的自然输出就是意图。

```
h_t → [World Model Head] → 预测下一步
    → [Action Head] → 行动（影响环境）
    → [Thought Head] → 内心自语（反馈给自身）
```

### 4. 持续学习 (Continuous Learning)
不只是推理时持续，训练也是持续的。
- 每个时间步都在更新
- 没有"训练集"和"测试集"的区分
- 模型从自己的经验流中实时学习

## 架构详情

```
            ┌──────────────────────┐
            │    Curiosity Gate    │ ← 预测误差高 → 好奇心门打开 → 加速学习
            └────────┬─────────────┘
                     │
┌──────────┐  ┌──────▼──────┐  ┌──────────┐
│ World    │  │ Recurrent   │  │ Action   │
│ Model    │◄─┤ Core (GRU)  │──► Head     │
│ (预测)   │  │ h_t = f(h)  │  │ (行动)   │
└──────────┘  └──────▲──────┘  └──────────┘
                     │
            ┌────────┴────────┐
            │  Embedding      │
            │ (obs + action)  │
            └─────────────────┘
```

### 参数规模

| 模块 | 参数 | 说明 |
|------|------|------|
| Token Embedding | 128 × 64 = 8,192 | 观察值编码 |
| Action Embedding | 16 × 16 = 256 | 动作编码 |
| GRU (256维) | 3 × (256² + 256×80 + 256) ≈ 258K | 循环核心 |
| World Model Head | 256 × 128 = 32,768 | 预测下一观察 |
| Action Head | 256 × 16 = 4,096 | 动作生成 |
| Curiosity Gate | 256 × 1 = 256 | 好奇心调节 |
| Thought Head | 256 × 128 = 32,768 | 内心语言 |
| **总计** | **≈336K** | 远小于 1.5B |

### 训练目标

```python
loss = world_model_loss           # 预测环境的准确性
     + curiosity_bonus            # 寻求高预测误差的状态
     + action_entropy             # 保持行动多样性
     + thought_coherence          # 维持"思考"的语义连贯性
```

## 环境

最小毒丸环境：

- 5×5 网格世界
- 每个格子有状态（空/墙/物体/特殊）
- 模型可以：↑↓←→ 移动 + 交互
- 观察：当前位置 + 相邻格状态 → 编码为 token
- 没有外在目标——唯一驱动力是好奇心

更简单 v0.1：**数字序列环境**
- 环境生成数字序列（有模式 + 随机噪声）
- 模型输出数字（影响后续序列）
- 目标：预测 → 好奇 → 探索规律
