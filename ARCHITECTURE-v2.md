# 词级思考链架构 — 设计

## 目标

从字符级升级到词级词表，让模型产生**有语义的思考链**：
```
🤔 "我想看看目录里有什么..." → 行动 ls
🤔 "输出是 total 8 drwxr-xr-x... 有文件" → 行动 cat
```

## Tokenizer

自定义 BPE tokenizer，在 Docker 命令输出文本上训练。

| 项目 | 值 |
|------|-----|
| 词表 | ~2000 tokens |
| 训练数据 | 所有命令的 Docker 输出文本 |
| 工具 | HuggingFace `tokenizers` 库 |

**词表布局**：
```
0-1999:  词 token（BPE 子词）
2000-2015: 行动 token（16 个命令）
2016-2023: 特殊 token（BOS, EOS, 等）
总计: ~2024 tokens
```

## 模型架构

| 模块 | 参数量 | 说明 |
|------|--------|------|
| Token Embedding | 2024 × 128 = 259K | 词 + 行动 + 特殊 |
| GRU (512) | 3 × (512² + 512×128 + 512) ≈ 990K | 循环核心 |
| LM Head | 512 → 2000 = 1.02M | 预测下一个词 |
| Action Head | 512 → 16 = 8K | 决定行动 |
| World Model | (512+128) → 2000 = 1.28M | 预测行动输出 |
| Curiosity Gate | 512 → 16 → 1 = 8K | 好奇心门 |
| **总计** | **~3.5M** | 10× 当前字符级 |

3.5M 参数 × fp32 ≈ 14MB — CPU 上完全可行。

## 思考链机制

```
循环:
  1. 观察: 命令输出 token 序列 → RNN 处理
  2. 思考: 模型生成词 token（= 内心独白）
  3. 行动: 模型输出行动 token → Docker 执行
  4. 预测: 世界模型预估行动结果
  5. 学习: 比较预测 vs 实际 → 更新参数
  
思考链的训练信号:
  - 世界模型预测越准 → 思考链被强化
  - 预测越不准 → 好奇心门打开 → 更多思考
  - 思考 → 行动 → 观察 形成一个完整周期
```

## 低成本的"低成本"

DeepSeek 的 GRPO 训练思考链需要大量算力。
我们的低成本方案：

```
1. 小词表 (2K vs 151K) → 小 embedding
2. 小模型 (3.5M vs 1.5B+) → CPU 可训
3. 在线学习 (不需要离线数据集)
4. 单任务域 (命令行 → 词表聚焦)
```
