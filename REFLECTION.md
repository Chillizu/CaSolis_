# 复盘：Folunar 走到哪了

## 我们建了什么？

一个**感知 → 分类 → 模板执行 → 奖励 → 学习的反射弧**：

```
StateText → Classifier → Intent → Template → bash → Output → Reward → Train
```

它工作，但它是一个**机械的、无意识的反射机器**。

## 七个局限

### 1. 8 个意图 = 8 种可能的"想法"

分类器只能在 8 个盒子里选一个。它没有"想新东西"的能力。真正的自主 Agent 应该能扩展自己的意图空间——发现新的可能行为、创建新的认知分类。

**现状**：8 选 1。没有第二层思考。没有 "这个问题不属于任何已知类别 → 创建新类别" 的机制。

### 2. 模板引擎 = 紧身衣

每个意图映射到固定 bash 命令。Agent 不能自由键入任何 Linux 命令——它只能从模板表里挑。

**现状**：Agent 永远无法发现 `find / -name "*secret*"` 或 `strace -p 1234` 或任何训练数据里没有的命令。

原始项目的 46M ModelCluster V3 曾试图从零生成命令文本——它失败了（能力不够、数据太少）。但模板引擎走到了另一个极端：零创造力。

### 3. RND 不驱动探索

RND 新颖度在前 100 步内降到 ~0.0002。原因：
- 状态空间太小（8 意图 × 14 目标 × 几个目录 = 很快被穷举）
- 预测器网络在 MLP 层迅速学会目标网络的输出
- 没有任何机制推动 Agent 去**寻找**新颖状态——它只在熟悉的地方打转

RND 的数学假设是 "从未见过的输入 → 高预测误差"。但如果 Agent 从不主动去没见过的地方，RND 永远不会触发。

### 4. 目标是硬编码的

14 个预定目标轮转。Agent 不设定自己的目标，不从输出中学习什么值得探索，不改变方向。

**现状**：Agent 没有"好奇心"——它在执行一个固定的目标队列。

### 5. 没有世界模型

Agent 不看状态理解因果关系。它不记得 "上次我在 /etc 做了 grep root，找到了 3 行匹配"。它没有"如果我做 X，可能发生 Y"的预测能力。

**现状**：单步反射。没有预测。没有计划。没有"如果-那么"推理。

### 6. 在线训练 = 强化已有模式

在线训练不教 Agent 新东西——它强化 Agent 已经在做的事情。当 Agent 开始偏爱 INFO（因为奖励高），训练让它更偏爱 INFO。这是正确的强化学习，但它是**收敛性的**，不是**扩展性的**。

### 7. 没有元认知

Agent 不思考自己的思考。它不评估 "我现在做的事情有价值吗？""我是不是在重复？""我换个方向会不会学到更多？"

**现状**：纯反射。感→知→动。没有"停下来想想"的环节。

---

## 但这些都是已知的

回头看这个项目的历史：
- 最初试图用 46M 从零模型同时做思考和生成 → 失败（数据和容量不够）
- 然后分拆 Brain（分类器）+ Hand（模板引擎）→ 能跑了，但失去了生成能力
- 现在有了一个可运作的反射弧 → 但它是一个**小圈子**

**真正的瓶颈不是精度（92% 够了），不是成功率（88% 够了），而是表征的封闭性。**

Agent 的所有可能行为都在 8 个预定义模板里。它永远不会做任何训练数据之外的事。

---

## 如果要从这里继续突破

### 路径 A：开放行为空间
不要 8 个模板，而是让 Agent 能自由构建命令。可以结合：
- ModelCluster V3 的文本生成能力（产生候选命令）
- 安全执行沙箱（Docker）
- 分类器判断 "这个命令看起来合理吗？"
- RND 评估 "这个命令会带我去新颖的状态吗？"

### 路径 B：开放认知空间  
不要 8 个固定意图，而是让意图系统本身可扩展：
- 当输入不在任何已知意图的高置信度区间时 → 创建新意图
- 新意图通过聚类 state_text 嵌入自动生成
- 新意图的模板从成功执行的命令中提取

### 路径 C：加入世界模型
- 在状态编码器上搭一个预测头：给定 state_text 和 intent，预测 output
- 输出预测误差 = 世界模型的 surprise → 好奇心信号
- 多步推理：在状态空间中做 beam search

---

## 但先回答你的问题

> 是否是局限的，是否是机械的？

**是局限的**。8 个意图 × 14 个目标 × 8 个模板 = 896 种可能的轨迹。穷举这个空间不需要多久。Agent 没有超越这个边界的机制。

**是机械的**。它是"如果 state_text 包含 'cpu' → 预测 INFO → 执行 cat /proc/cpuinfo"。没有真正的选择——只有概率性的模式匹配。这不是"思维 = 概率 + 最优选择"中那个"选择"的含义——选择应该是**有意识的、有目标的**，而不只是 softmax argmax。
