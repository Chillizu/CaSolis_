## 当前架构（已实现）

```
OnlineAgent.step()
  │
  ├─ 50% → Conductor → 置信度 > 0.8?
  │     ├─ ✅ 是 → Nanny翻译 → 模板引擎 → Docker执行
  │     └─ ❌ 否 → 降级到旧分类器
  │
  └─ 50% → 旧分类器 (稳定基线, 11类, 91%)
         ↓
       模板引擎 → Docker执行
```

## 推进计划（Kimi 优先级）

### P0: 降低 A/B 阈值（立即）

**现状**: 阈值 0.8 太保守, 把训练好的 Conductor 锁在门外。

**实验方案**: 阈值网格扫描

| 阈值 | 预期使用率 | 预期成功率 | 决策 |
|------|-----------|-----------|------|
| 0.8 (当前) | ~20% | ~47% | 基线 |
| 0.7 | ~35% | ~45% | 如果成功率 > 40% 可取 |
| 0.6 | ~50% | ~40% | 如果使用率翻倍, 接受轻微下降 |

每个阈值跑 100 步, 记录使用率、成功率、HELP 收敛速度。

### P1: 多命令组合（近期）

**现状**: 保姆只翻译成单条命令。

**目标**: 保姆返回 `["cmd1", "cmd2", "cmd3"]` 命令序列, 顺序执行。

**方案**:
1. 模板层扩展: 每个意图关联 2-3 条命令
2. 想法向量的高激活维度作为"探索深度"信号
3. 深度高 → 执行更多命令
4. 深度低 → 只执行 1 条

```python
# 扩展后的意图模板示例
INTENT_MULTI_COMMANDS = {
    "INFO_NETWORK": ["cat /etc/hosts", "ip addr", "ss -tlnp"],
    "INFO_PROCESS": ["ps aux", "pstree", "uptime"],
    "READ_CONFIG":  ["cat /etc/passwd", "cat /etc/group", "cat /etc/shadow"],
}
```

### P2: Conductor 在线训练（条件触发）

**触发条件**: 指挥家使用率稳定 > 40% 且平均奖励不再下降。

**方案**:
1. 从经验缓冲区采样 Conductor 路径的 `(state, intent, reward)` 三元组
2. 用 reward 加权训练 (高奖励样本权重高)
3. 小学习率 (1e-5), 防止灾难性遗忘
4. 定期回滚到最佳 checkpoint

### P3: 知识状态跟踪（辅助模块）

不单独做, 作为在线训练的辅助:
- 未知状态标记为高好奇心样本
- 优先加入经验回放
- 与 RND 信号融合

### P4: 保姆 LLM 升级（未来）

在 Conductor 使用率饱和 + 多命令能力见顶后, 再考虑 Gemma4。

**前提条件**:
- Conductor 使用率 > 60%
- 模板层无法覆盖的新情况积累 500+
- Gemma4 本地输出稳定性验证通过
