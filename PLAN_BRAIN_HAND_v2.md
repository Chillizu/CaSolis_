# CaSolis_ v2 — 大脑+手脚架构（修正版）

> 基于四大模型（ChatGPT · Claude · Gemini · Kimi）的评审修正

---

## 三个核心修正

### 修正 1: Qwen 不是"bash 翻译器"，是"参数推理器"

```
v1 的误解: Qwen 的工作是把意图→bash（模板填充 → 浪费）

v2 的正确分工:
  大脑 (46M / 预训练 backbone): 选意图（做什么）
  Qwen (1.5B): 推理参数（怎么做）
    例: 意图=GREP → Qwen 看当前状态 → "pattern 应该搜 'root'"
    例: 意图=READ_FILE → Qwen 猜路径 → "可能是 /etc/passwd"
  模板引擎: 确定性填充（0 幻觉, 0 延迟）
    例: GREP pattern=root → grep 'root' /etc/passwd
```

这样 Qwen 不是在填充模板，而是在**真正推理**——它需要知道文件系统结构、命令参数含义、当前上下文。这才是小模型做不到的部分。

### 修正 2: 从零训 46M → 预训练 backbone + 轻量头

```
v1: 46M ModelCluster 从零训在 2000 条数据上
    Kimi 说: "参数/数据比 23,000:1, 必然过拟合"
    ChatGPT 说: "BLAS 在千级样本上几百条就能~90%"

v2: 用 Sentence Transformer (all-MiniLM-L6-v2, 22M) 做 backbone
    冻结主干, 只训分类头 (20→10), ~50K 可训参数
    数据 2000 条对于 50K 参数是合理的
```

这不是"放弃从零训练"——大脑的核心部分（路由器、世界模型、评估器）仍然是我们的。只是把**文本表示**这一层交给预训练模型。就像人用眼睛看世界——眼睛不是自己长的，但怎么用眼睛想是自己的。

### 修正 3: ICM（预测误差好奇心）→ RND（固定随机网络蒸馏）

```
v1: 世界模型预测输出嵌入 → 误差高 = 好奇
    问题: date/ps/uptime 永远预测不准 → Noisy TV

v2: 固定随机网络 (target) + 可训练预测器 (predictor)
    目标不是预测输出，而是预测 random network 对输出的编码
    对于 date: 随机网络的输出也是随机的 → 预测器永远学不会 → 误差一直高
    对于 ls /etc: 输出是确定性的 → 预测器能学会 → 误差下降
    不对！RND 的原理是：如果状态确定，随机网络输出也是确定的，预测器能学会
    如果状态随机，随机网络输出也是随机的，预测器学不会 → 误差高
    但 RND 的核心是"可学习的确定性" → 对于 date，其输出文本是随机的，
    但随机网络对随机输入也有随机输出 → 预测器无法降低误差
    
    等等，让我想清楚 RND 的核心机制...
    
    RND: target network f* (固定随机) + predictor network f (可训练)
    对同一状态 s, f*(s) 和 f(s) 都输出固定向量
    环境随机性: date 的输出 text 每次不同 → 我们需要的是 "对 text 的编码"
    
    如果 f* 应用到 text 本身: f*(text_date1) ≠ f*(text_date2)
    预测器 f(text_date1) 可以学会 f*(text_date1) → 误差小
    这不对，没有解决问题...
    
    正确的 RND 做法是:
    RND 不应用于"预测输出嵌入", 而是应用于"状态编码"
    状态: [当前目录, 最近执行结果, 已学意图...]
    状态本身如果确定, f*(状态) 确定 → 预测器能学会
    date 命令会改变状态? 不, 它只产生输出, 不改变状态
    所以对于 date, 状态不变 → f*(状态) 固定 → 预测器学会 → 误差小 → 不好奇
    对于探索新目录: 状态变 → f*(新状态) 新 → 预测器还没学 → 误差大 → 好奇

    对！RND 的核心是预测"下一个状态的编码", 不是"输出嵌入"
```

> 上面的推导过程保留以展示思考链路。结论: RND 应该应用于状态编码转移, 而不是输出嵌入预测。

```
v2 修正后:
  状态 = (当前目录, 已访问路径集合, 最近 N 步记录, 已学意图列表)
  RND target: 固定随机网络 f*, 对状态编码输出 R^d
  RND predictor: 可训练网络 f, 学习预测 f*(状态')
  好奇心 = ||f(状态') - f*(状态')||^2
  如果状态'是已知模式 → 预测器能猜到 → 不好奇
  如果状态'是全新的 → 预测器猜不到 → 好奇
  注意: date 命令不改变状态 → 不会触发好奇 ✅
        ps aux 不改变状态 → 不会触发好奇 ✅
        第一次 ls /etc/ssl → 状态变了 → 准确好奇 ✅
```

## 训练策略

### 意图分类头
- backbone: all-MiniLM-L6-v2 (冻结, 22M)
- 头: Linear(384, 10), ~4K 参数
- 数据: 2000 条 (4K 参数 / 2000 条 = 2:1, 合理)
- 训练: 10 epoch, 1 min/epoch, CPU

### 世界模型 (RND)
- RND target: 固定随机 2-layer MLP (384→256→256)
- RND predictor: 可训练 2-layer MLP (384→256→256, ~200K 参数)
- 好奇心信号: predictor 和 target 对*下一个状态*编码的 MSE
- 训练: 在线, 每 N 步更新 predictor
