# CaSolis_ 最终架构计划 — 自生长专家集群

## 核心理念

模型不是「一个会推理的大脑」，而是「一群专家 + 一个路由器 + 一个世界模型」的生态系统。

```
核心假设：
  「思考」= 多路并行可能 → 选最优 → 验证 → 意外驱动探索

这不是 Transformer/Mamba 的变体。
这是一个「认知系统架构」——模型只是其中一部分。
```

## 架构总览

```
                          ┌──────────────┐
                          │  外部世界     │ (Docker 沙箱)
                          └──────┬───────┘
     执行命令 ← ────────────┐    │ 观察输出
                            ↓    ↓
                     ┌──────────────────┐
                     │   路由器          │
                     │   (小 MLP)       │
                     │   输入: 观察 +    │
                     │   世界模型误差     │
                     └──┬──┬──┬──┬─────┘
                        │  │  │  │
               ┌────────┴──┴──┴──┴────────┐
               │                          │
         E0 E1 E2 E3 ...          En (新专家)
         (现有)                  (动态创建)
               │                          │
               └──────────┬───────────────┘
                          ↓
                    ┌────────────┐
                    │ 世界模型    │ ← 预测输出嵌入
                    │ (共享头)    │    预测误差 = 意外/好奇信号
                    └──────┬─────┘
                           ↓
                     预测误差 > 阈值？
                    ┌────┴────┐
                    ↓         ↓
                 加新专家    继续探索
```

## 三个核心机制

### 1. 路由分工（无监督涌现）

所有专家从相同 checkpoint 初始化，但路由器在训练中自然学会：
- `ls` 类 → E0
- `cat` 类 → E2
- 不常见命令 → E1（创造专家）
- 需要评估 → E3

这是 MoE 的标准路由训练 —— 不需要手动标签。

### 2. 世界模型驱动的好奇心

```
世界模型（共享 nn.Linear(d_model, d_model)）

作用：
  ┌ 给定隐藏状态 → 预测输出嵌入
  ├ MSE 误差 = 世界模型对「这个命令会产生什么输出」的不确定度
  └ 误差大 = 意外 → 路由器切换到创造/探索模式
```

好奇心的实现：
```
if world_error > threshold:
    router.temperature += exploration_noise
    # 路由器更可能选 E1（创造专家）或触发加新专家
```

### 3. 动态加专家

新专家什么时候创建：
```
当路由器发现「所有现有专家在这个输入上的预测误差都高」：
  1. 世界模型看：所有专家的预测误差都 > 阈值
  2. → 这是一个新领域，没有专家专精于此
  3. → 创建新专家 En，从当前数据初始化
  4. → 路由器学会在类似输入时选 En
```

新专家不会让旧专家遗忘——旧专家的权重不更新。

### 4. 自增长命令池

```
Docker 验证 → 有效命令加入已知池
有效命令的变体 → 实验性加入
无效命令 → 记录为「不要做的事」

命令池增长 = 数据自动积累
数据积累 = 新专家有了训练素材
```

## 区别于传统 LLM 的关键点

| 传统 LLM | 本架构 |
|----------|--------|
| 一个模型会所有事 | 多个专家各管一摊 |
| 学到新模式 = 覆盖旧模式 | 加新专家，不动旧的 |
| 能力天花板 = 模型大小 | 能力天花板 = 专家数量 |
| 无法解释「为什么选这个输出」 | 路由器选择可观察（专家 ID + 权重） |
| 好奇心 = 随机采样 | 好奇心 = 世界模型误差驱动 |

## 想请 Kimi 评审的问题

1. **动态加专家的可行性** — 是否已有类似研究（MoE 动态扩展）？
2. **世界模型作为好奇心驱动的有效性** — 预测误差真的能驱动有意义的探索吗？还是会产生噪声？
3. **专家数量上限** — 单个模型里能塞多少个 7.4M 的专家而不退化？
4. **遗忘问题** — 旧专家不更新 = 不会遗忘，但新专家学到的和旧专家的知识怎么协调？
5. **CPU 训练效率** — 随着专家增多，训练时间线性增加。有优化空间吗？
6. **整体评价** — 这个架构的致命缺陷是什么？最可能失败的点？
